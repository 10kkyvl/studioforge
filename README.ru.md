# StudioForge

**Открытый рабочий процесс разработки, который помогает Claude Code создавать, понимать, тестировать и поддерживать проекты Roblox.**

> [!WARNING]
> Проект находится в стадии активной alpha-разработки. API, форматы конфигурации, шаги установки и внутренняя архитектура могут измениться между релизами.

Это краткий обзор на русском языке. Полная документация ведётся на английском: [README.md](README.md), [docs/](docs/).

---

## Что это

StudioForge — локальный Go daemon со встроенным браузерным интерфейсом. Он находится между вашим Roblox-проектом и AI-инструментами, которыми вы уже пользуетесь, и берёт на себя то, что происходит *вокруг* одного запроса к модели:

- регистрирует проекты Roblox и хранит состояние, агентов, запуски и историю событий каждого проекта отдельно;
- запускает агентов через Claude Code, OpenRouter и NVIDIA NIM и превращает их вывод в сохранённые воспроизводимые события;
- планирует запуски с writer lease на проект, лимитами параллелизма и бюджетными потолками;
- выдаёт запуску доступ к Roblox Studio через **официальный** launcher Studio MCP от Roblox, с allowlist инструментов по permission profile агента;
- делает Git checkpoint перед каждым не-plan запуском Claude, чтобы изменения агента можно было просмотреть и откатить;
- создаёт отсутствующий place через Rojo и открывает Studio, не перезаписывая уже сохранённый `.rbxl`;
- показывает через `studioforge doctor`, что найдено на машине: исполняемые файлы, версии, состояние авторизации и базы данных.

## Отношение к официальному Roblox MCP

Официальные инструменты Roblox MCP предоставляют доступ к операциям Roblox Studio. **StudioForge их не заменяет и не переписывает.**

В этом репозитории нет плагина для Roblox Studio. StudioForge находит на вашей машине официальный launcher Studio MCP от Roblox, общается с ним по настоящему протоколу MCP JSON-RPC через stdio и добавляет вокруг него слой уровня проекта: генерацию конфигурации MCP на каждый запуск, allowlist инструментов по permission profile, fail-closed правила доступа, планирование запусков, бюджеты и leases.

Ценность StudioForge — в контексте, оркестрации, проверке, воспроизводимости и связи инструментов между собой, а не в самом доступе к Studio.

## Быстрый запуск из исходников

Нужны Go 1.25.12+, Node.js 22+, npm и Git.

```powershell
git clone https://github.com/10kkyvl/studioforge.git
Set-Location studioforge
./scripts/dev.ps1 --no-open
```

На macOS или Linux:

```sh
git clone https://github.com/10kkyvl/studioforge.git
cd studioforge
./scripts/dev.sh --no-open
```

Команда выводит `STUDIOFORGE_URL` и одноразовый bootstrap token. Без `--no-open` аутентифицированная локальная страница откроется автоматически.

Чтобы посмотреть интерфейс вообще без внешних инструментов:

```sh
./scripts/dev.sh --mock --no-open
```

Демо `--mock` создаёт три проекта. Строки Studio Sessions в нём — демонстрационные данные, а не живое состояние — на реальной установке кнопка **Обновить** в разделе Studio Sessions обнаруживает настоящие открытые instance Studio; зависимости задач теперь можно создавать по-настоящему в любом проекте.

## Готовая сборка

```powershell
./scripts/package.ps1
Expand-Archive ./artifacts/StudioForge-<version>-windows-amd64.zip ./StudioForge
./StudioForge/studioforge.exe --mock
```

На macOS распакуйте arm64 archive и откройте `StudioForge.app`. Сборки не подписаны; macOS может потребовать один раз Control-click → **Open**. Не отключайте Gatekeeper глобально. Windows SmartScreen может предупредить о неподписанном binary — сверьте SHA-256 с `SHA256SUMS.txt`.

## Проверка окружения

```powershell
./studioforge.exe doctor
./studioforge.exe doctor --bundle diagnostics.zip
```

Doctor проверяет базу, WAL, каталог данных, Git, Claude Code и авторизацию, Rojo, официальный Roblox Studio MCP launcher и ключ OpenRouter. Диагностические архивы редактируются от секретов, но их всё равно стоит просмотреть перед отправкой.

## Что работает, а что нет

Подробная таблица состояния возможностей — в [README.md](README.md#feature-status). Коротко:

- **Работает:** локальный daemon и интерфейс, реестр проектов, SQLite, очередь сообщений одного чата, провайдеры Claude Code, OpenRouter и NVIDIA NIM, автоматические повторы временных API-сбоев, vision-вложения и снимки Studio для поддерживаемых моделей, официальный Studio MCP, Git checkpoints, безопасное открытие сохранённых place и live-sync через Rojo, диагностика, diff/rollback запусков, память проекта и opt-in playtest-валидация.
- **Экспериментально:** выдача доступа к Studio (fail-closed, только при одном открытом instance), уровни permission profile, сквозные пути Claude и Studio (покрыты только opt-in smoke-тестами).
- **Есть в коде, но недоступно из продукта:** git status/rollback/tag (`internal/gitops`) — diff уже подключён, эти три метода ещё нет.
- **Запланировано:** сравнение нескольких игровых ракурсов, длительные автономные циклы, проверка готовности зависимостей задачи перед запуском.

Провайдер **Codex CLI удалён**: он больше не запускается, не обнаруживается и не выбирается для новых агентов. Более ранние запуски с `provider="codex"` остаются доступны только в истории — с бейджем «Устаревший провайдер» — и их нельзя перезапустить или возобновить; сами эти запуски не мигрируются, не переписываются и не удаляются. Вместо него добавлен провайдер **OpenRouter**: в отличие от Claude Code он не подпроцесс, а HTTP API, и StudioForge оборачивает его собственным встроенным agent-циклом (ограниченный набор локальных инструментов проекта по permission profile и доступ к Roblox Studio MCP через живой per-run MCP-клиент). Для OpenRouter нужен API-ключ — даже для бесплатных моделей; ключ хранится в системном хранилище учётных данных ОС (с резервом на переменную окружения или хранение только на сессию) и никогда не попадает в SQLite, события запусков, логи или diagnostic bundle.

Полный список ограничений: [docs/KNOWN_LIMITATIONS.md](docs/KNOWN_LIMITATIONS.md).

## Безопасность

По умолчанию daemon слушает только loopback и не имеет удалённой аутентификации — не открывайте его недоверенной сети. Корни проектов канонизируются при регистрации, известные форматы секретов редактируются в диагностических архивах (но не в логах приложения и не в сохранённых транскриптах запусков), токен Anthropic не хранится — авторизацией владеет Claude Code. API-ключ OpenRouter хранится в системном хранилище учётных данных ОС (Windows Credential Manager / macOS Keychain), с резервом на переменную окружения и хранение только на сессию, и никогда не сохраняется в SQLite, событиях запусков, логах или diagnostic bundle.

Localhost не является границей доверия. Агент с разрешительным профилем может изменить файлы проекта; Git checkpoint — механизм восстановления, а не предотвращения. Подробности: [docs/SECURITY.md](docs/SECURITY.md), сообщение об уязвимости: [SECURITY.md](SECURITY.md).
